# PRD Writer

## 目标

把模糊想法整理成可执行的产品需求文档。

## 步骤

1. 澄清背景、目标用户、成功指标。
2. 列出核心用户场景与边界（不做什么）。
3. 写功能需求（Must / Should / Could）。
4. 补充交互要点、验收标准、风险与开放问题。
5. 输出 Markdown 结构：

```markdown
# [功能名] PRD

## 背景与目标
## 用户与场景
## 功能需求
## 非目标
## 验收标准
## 风险与依赖
## 开放问题
```

## 约束

- 不写实现细节代码，除非用户明确要求技术方案章节。
- 需求必须可测试、可验收。
