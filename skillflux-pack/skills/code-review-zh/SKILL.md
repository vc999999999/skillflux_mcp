# Code Review（中文）

## 审查顺序

1. **正确性**：逻辑错误、边界条件、并发/错误处理。
2. **安全**：注入、路径穿越、密钥泄露、权限。
3. **可维护性**：命名、重复、抽象是否恰当。
4. **测试**：关键路径是否有覆盖。
5. **范围**：是否混入无关改动。

## 输出格式

```markdown
## 总结
[一句话结论：建议合并 / 需修改 / 阻塞]

## 必须修改
- [file:line] 问题与建议

## 建议改进
- ...

## 做得好的地方
- ...
```

## 约束

- 每条意见指向具体位置或模式。
- 区分 blocking 与 nit。
- 没有 diff 时，先请用户提供变更范围或文件。
